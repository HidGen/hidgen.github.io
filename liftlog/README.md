# Подходы — лёгкий офлайн-дневник качалки

PWA на ванили (HTML/CSS/JS) + IndexedDB. Без сборки, без бэкенда: вся история
тренировок хранится **на устройстве**, работает офлайн. Интерфейс — под усталые руки:
крупные кнопки, никакой клавиатуры, новый подход по умолчанию повторяет прошлый.

## Запуск
Нужен любой статический сервер (для Service Worker — контекст `localhost` или `https`).
```bash
cd liftlog
python -m http.server 8080
# открыть http://localhost:8080
```
На телефоне для полноценного PWA (установка на экран, офлайн-кэш) нужен **HTTPS**
(например, GitHub Pages). По чистому HTTP приложение тоже работает (IndexedDB доступен),
но без установки и офлайн-кэша оболочки.

## Флоу
1. **Начать тренировку**.
2. **+ Упражнение** → имя (чип из недавних или ввод) + стартовый вес.
3. На экране упражнения: степперы веса (±2.5) и повторов (±1), большая кнопка
   **«Записать N × R»**. Тот же подход — один тап; изменил — пара тапов по ±.
4. **Готово** → упражнение в списке тренировки. **Завершить тренировку** → в Историю.

## Контракт хранения (IndexedDB `liftlog`, расширяемый)
Упражнение имеет **тип `kind`**, а подход — открытую карту **`values`**, поэтому модель
не привязана к «вес/повторы». Типы и их поля описаны в реестре `KINDS` (в `app.js`) —
добавить новый тип (плавание, велотренажёр…) = дописать запись в реестр, без миграций.

- `workouts` {id, startedAt, finishedAt, …sync}
- `exercises` {id, workoutId, name, **kind**, startedAt, …sync}
- `entries` {id, exerciseId, idx, **values{}**, loggedAt, …sync}
- `catalog` {name, kind, lastValues, lastUsed} — память для чипов

Типы из коробки: `strength` (вес+повторы), `bodyweight` (повторы+доп.вес),
`cardio` (время, дистанция, ср.пульс, нагрузка), `timed` (время).

**Задел под синхронизацию (post-facto):** каждая запись несёт `id`(uuid), `userId`,
`updatedAt`, `deleted`(тумбстоун). Удаление — мягкое. Будущий синк: push «грязных»
(`updatedAt > lastSync`) + pull, слияние **last-write-wins**. До серверного синка —
**Экспорт/Импорт JSON** (в Истории): бэкап и ручной перенос между устройствами.
